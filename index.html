7 princípios do Meteor

  • Data on the Wire: Não enviar HTML pela rede. Enviar dados.
  • One Language: Escrever Javascript no cliente e no servidor.
  • Database Everywhere: Usar a mesma API para acessar o banco de dados no cliente e no servidor.
  • Latency Compensation: Usar prefetching e simulação de modelo para emular latência zero
  • Full Stack Reactivity: Fazer o tempo-real ser padrão.
  • Embrace the Ecosystem: Ser open source e integrar com outras ferramentas open source.
  • Simplicity Equals Productivity: A melhor maneira de parecer simples é ser simples.

Coleções

Do lado do servidor temos o famoso e desenvolvido MongoDB.

Do lado do cliente, seu filho, Minimongo, que ainda não cresceu tanto.

Em cima disso, criou-se variáveis Session, com persistência fraca utilizando o minimongo.

Reatividade

Na programação imperativa, fazemos a = b + c, onde a terá o valor imediato que b e c vale naquele momento. Se mudarmos b ou c, precisamos escrever novamente esta expressão para que a valha o valor da nova soma.

Na programação reativa, a vale a soma entre b e c logo que b ou c são alterados, sem precisamos reavaliar a expressão.
